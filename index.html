Safe economic agency on EVM.

Tunnel is a protocol pattern for giving AI agents bounded authority: policy commitments, signed state receipts, and settlement paths that remain auditable.

core thesis

Delegate policies, not private keys.

The result is a verifiable control plane for delegating bounded economic actions to autonomous software.

authorizationreceiptsettlement
Motivation

AI agents are becoming economic actors, but today’s delegation model is unsafe.

Deploying an agent currently means trusting a host with keys, sessions, prompts, and incomplete logs. Tunnel reframes delegation as a cryptographically bounded relationship: policies instead of private keys, receipts instead of blind trust, and audit trails that connect on-chain and off-chain actions.

hosting trust trap

Private-key custody

Hosting an agent often requires handing control to infrastructure that can act beyond the user’s intended mandate.

blind delegation

Web-session opacity

Legacy credentials and cookies let agents act on websites without producing verifiable evidence of policy compliance.

host interference

Replay and suppression

A host can suppress, replay, or selectively query outputs until it obtains a favorable action.

audit gap

Fragmented accountability

On-chain events, off-chain transcripts, and agent decisions are difficult to reconstruct as one coherent activity log.

Protocol model

Tunnel has three protocol moments.

The lifecycle has three phases: create a bounded delegation, update it through signed cumulative receipts, and settle through a dispute-aware close path.

Protocol role

Turn authorization into a signed envelope.

Tunnel begins by converting an open-ended trust relationship into a signed policy object that can be enforced and audited.

1
User commits to constraintsNot “use my key”; rather “you may perform these actions under these limits.”
2
Provider accepts those constraintsThe provider countersigns so later settlement has a shared root of truth.
3
Receipt v0 becomes the initial stateEverything after this is an update to a verifiable state machine.
Protocol role

Each micropayment is a state transition, not an isolated invoice.

Cumulative receipts compress many off-chain actions into the latest verifiable claim, reducing on-chain cost while preserving auditability.

1
Agent requests an actionThe provider checks that the policy envelope allows it.
2
Receipt updates cumulative spendOnly the latest valid receipt matters for settlement.
3
Off-chain fast pathMost work stays off-chain until close, keeping gas low.

Protocol role

Settlement provides the safety boundary.

The close path defines the protocol response to stale submissions, provider failure, and user exit.

1
Permissionless closeEither party with a valid receipt can close the tunnel.
2
Challenge stale stateNon-final receipts trigger a window where newer signed state supersedes older claims.
3
Deterministic splitUsed amount goes to provider; remaining deposit returns to user.

Use cases and scenario simulations

Bounded agency is useful when actions carry economic or reputational risk.

The same receipt-based control plane applies across consumer agents, developer APIs, and institutional automation: define limits, execute off-chain, and retain a verifiable path to settlement or dispute.

consumer agent

Autonomous subscription manager

A user authorizes an agent to negotiate or cancel recurring web services, but only within a spending cap, approved merchant list, and expiration window.

Policymonthly cap, allowed domains, no long-term commitments
Actionagent contacts provider and approves a discounted renewal
Evidencereceipt links API action, amount, and policy hash
developer API

Pay-per-use AI inference

Developers call an Anthropic-compatible endpoint while usage is metered against a funded tunnel. The API key authenticates requests without granting custody over the wallet.

Policymodel tier, rate limit, maximum daily spend
ActionPOST /v1/messages through 3mate proxy
Evidencecumulative receipt records token usage and balance impact
institutional workflow

Auditable procurement agent

An organization lets an agent purchase approved services, while finance and compliance can reconstruct what was requested, authorized, delivered, and settled.

Policyvendor allowlist, budget envelope, approval threshold
Actionagent orders data/API services within the envelope
Evidencereceipts plus on-chain close produce an audit trail
3mate platform API architecture

Tunnel becomes a pay-per-use AI API proxy.

3mate already demonstrates the product shape: users open a Sui Tunnel, register an Ed25519 API key, call an Anthropic-compatible endpoint, and the platform meters usage against tunnel balance.
