Safe economic agency on EVM.

Tunnel is a protocol pattern for giving AI agents bounded authority: policy commitments, signed state receipts, and settlement paths that remain auditable.

Delegate policies, not private keys.

How to explain it

The settlement slide is the security argument.

This is where EF reviewers will care: what happens when someone submits old state, when the provider disappears, or when the user wants out?

1
Permissionless closeEither party with a valid receipt can close the tunnel.
2
Challenge stale stateNon-final receipts trigger a window where newer signed state supersedes older claims.
3
Deterministic splitUsed amount goes to provider; remaining deposit returns to user.
Methodology

A 12-month path from protocol design to public Ethereum infrastructure.

M1 · 1–3

Formal spec + Solidity

State machine, security properties, ERC-4337 integration, Foundry tests, gas benchmarks.

M2 · 4–6

Audit + TLSNotary

Event indexing, attestation anchoring, SDK, and unified trace reconstruction.

M3 · 7–9

Host-interference analysis

Replay, suppression, selective querying, challenge windows, and containment sidecars.

M4 · 10–12

Integration + publication

ERC-8004 demo, open-source release, evaluation, FC/AFT submission.
